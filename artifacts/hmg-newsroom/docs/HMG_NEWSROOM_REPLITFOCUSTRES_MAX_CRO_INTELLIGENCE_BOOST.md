## CRO Review — 8 Sections

Every Max CRO Review (generated locally by `maxCROEngine.ts`) produces:

1. **Sponsor Angle** — What brand/category might care about this content or audience?
2. **Relationship Follow-Up** — Who might need a call, email, DM, or warm follow-up?
3. **Content-to-Revenue Move** — How can this article/clip/social post become a sales asset?
4. **Brand Partnership Idea** — What HMG vertical or advertiser category fits this?
5. **Offline Money Play** — Event, interview package, local sponsorship, merch, consulting, live activation, or media package?
6. **What To Ignore** — What is not worth chasing?
7. **Founder Next Move** — One clear action for the Founder.
8. **Risk / Reputation Note** — What could make the revenue play look corny, desperate, unsafe, or off-brand?

## Statuses

| Status | Meaning |
|---|---|
| `Revenue Review Needed` | Revenue signal detected, waiting for Max review |
| `Max Review Drafted` | Local CRO review complete, ready for Founder |
| `Founder Review Required` | Flagged for Founder decision before any action |
| `Saved to Output History` | Brief saved to Output History log |
| `Relationship Follow-Up Needed` | Flagged for relationship tracking |
| `Ignore / No Money Move` | No revenue signal or manually marked ignore |

## Actions

| Action | Result |
|---|---|
| Send to Max | Runs `maxCROEngine.runMaxCROReview()` — local deterministic review |
| Copy Max Revenue Brief | Copies full 8-section brief to clipboard |
| Save Max Revenue Brief | Saves to Output History as `max-cro-brief` kind |
| Save to Output History | Alias for above — updates status |
| Mark Follow-Up Needed | Sets status → `Relationship Follow-Up Needed` |
| Flag: Founder Review | Sets status → `Founder Review Required` |
| Mark Ignore | Sets status → `Ignore / No Money Move` |
| Delete | Removes item from inbox |
